🔴 Pontos Críticos Identificados
1. Skill Engine - Inconsistência de Formatos
Problema: Skills usam 2 formatos diferentes:

Formato antigo (simples):

# docker
- **Repo:** https://...
- **Description:** docker tool
- **Install:** dnf install docker -y
- **Commands:** docker
- **Type:** tool
Formato novo (YAML frontmatter + markdown):

---
name: proton
category: security
install:
  rocky_linux:
    - "echo 'Instale...'"
commands:
  status: ["protonvpn-cli status"]
---
Impacto: Parser em internal/skill/manager.go:38-56 só lê formato antigo. Skills como proton.md não funcionam corretamente.

2. Hardcoding de Credenciais
CRÍTICO - cmd/nux/commands/proton.go:132-136:

Credenciais comentadas no código = violação de segurança.

Custom error types
